# Liver-Segmentation-3D-Reconstruction

 Сегментация печени по КТ-снимкам и 3D-реконструкция

Данный проект представляет собой полностью автоматизированный **End-to-End пайплайн** для сегментации тканей печени на основе сырых данных компьютерной томографии (DICOM). Система использует нейросети с архитектурой Instance Segmentation и генерирует готовые к 3D-печати или клиническому анализу твердотельные модели (`.stl`), исключая необходимость использования тяжеловесного стороннего ПО (например, 3D Slicer).

## 🚀 Ключевые особенности и инновации
- **Высокоточная модель v2:** Проблема переобучения (overfitting) была решена с помощью аугментации данных и нормализации по шкале Хаунсфилда (HU). Достигнутая точность **mAP@0.5 составляет 92.4%**, что превышает стандартные медицинские требования.
- **Отказоустойчивая архитектура (Retry Mechanism):** Внедрен механизм автоматического повтора (try-except + time.sleep) для защиты потоковой обработки сотен срезов от разрывов SSL-соединений и таймаутов API.
- **Продвинутая геометрическая фильтрация:** На этапе применения алгоритма Marching Cubes интегрирован фильтр площади (`cv2.contourArea`) для удаления шумов и оптимизация Z-Spacing (`spacing=0.3`) для коррекции вертикального искажения КТ-аппарата.

## Метрики производительности модели
Модель демонстрирует высокую стабильность при отделении тканей печени от фона, минимизируя уровень ложноположительных (False Positive) срабатываний.

- mAP@0.5 (Эквивалент коэффициента Сёренсена-Дайса): 92.4%

- mAP@0.5:0.95 (Строгий IoU): 74.8%

- Скорость инференса: ~0.015 сек на один КТ-срез.


## Установка зависимостей и запуск

Поскольку основной пайплайн реализован в формате Jupyter Notebook, перед началом работы необходимо установить требуемые библиотеки. Вы можете сделать это прямо в первой ячейке ноутбука.

Стандартная установка через ячейку Jupyter:
```text
!pip install pydicom roboflow opencv-python numpy scikit-image trimesh
```
Альтернативная установка (Рекомендуется):
Если стандартная команда !pip выдает ошибки путей окружения (environment path issues) в Jupyter, используйте модуль sys для точной установки зависимостей в текущее ядро:
```text
import sys
!{sys.executable} -m pip install pydicom
!{sys.executable} -m pip install trimesh scikit-image roboflow opencv-python numpy
```
